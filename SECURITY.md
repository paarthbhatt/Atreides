# Security policy

Atreides is a security demonstration. Do not use its synthetic tools or fixtures to test systems you do not own or have permission to assess.

## Reporting a vulnerability

Do not open public issues for a suspected vulnerability. Use GitHub's private vulnerability reporting flow for the published repository, including a minimal reproduction, impact, affected commit, and suggested mitigation where possible.

## Scope and limitations

The current build is intentionally a local demonstration: receipts are in memory, the controlled upstream tools are simulated, and the HTTP gateway has no authentication layer. It must not be represented as production-ready security control software.
